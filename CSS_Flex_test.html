px ( absolute )

Avoid using 'px' for font-size
Use mostly for small details like border and shadow.

  • fixed in size
  • not responsive
  • overrides user's browser perferences

% ( relateive )

I recommend using percentages for layouts and width.height.
For example, laying out links on havbar, placing images inside a div, ets.

  • size is defined as percentage of another value ( mostly parent element )
  • somethimes size is defined as percentage of the element itself

em ( relateive )

You can use 'em' for font-size and margin/padding.
Use it when you want to adjust margin/padding based on that element's font-size
( if font-size is big, you maybe want bigger spacing. )

  • changes behavior based on property.
  • 1 em == parent font-size
  • If parent doesn't have a size, defaults to 16px (boxy)

rem ( relateive )

You can also use it for font-size and margin/padding
It is easier to work with than 'em' because it's more consistent.

  • relative to root HTML, no matter what (default is 16px)
  • you can change the root HTML, size. for example, if you change it to 20px, 1 rem will always be 20px

vw/vh ( relateive )

vw/vh are relative to the width/height fo the browser window
100vw means full width of the screen. Use vw/vh for bigger layouts, like background.

  • useful for responsive wwebsites because everything scales.

ch

It is relative to the width of the number 0 of the current font.
'ch' is used to size the width of a paragrach. In general, you want a 45-70 character wide column for readability.

  • You would use it likt this : max- width 40ch;
  • This sets the width of the column to ad maximum of 40 characters per line.
